Demo Architecture: 2-3 Approaches

How should we organize the Ch 2-3 interactive demos?

A

Integrated Playground (All-in-One)

Ch 2: Backward Pass Debugger + Derivative Visualizer in one space with shared examples

Ch 3: Single "Optimizer Playground" with loss landscape, weight updates, convergence race, and metrics all visible together

Pros: Holistic learning, see relationships between concepts. Cons: Complex UI, may overwhelm at first.

B

Modular Tabs (Focused Views)

Ch 2: Tab 1: Backward Pass Debugger | Tab 2: Derivative Visualizer

Ch 3: Tab 1: Loss Landscape Explorer | Tab 2: Weight Update Viz | Tab 3: Convergence Race | Tab 4: Optimizer Tuner

Pros: Clean, focused. Easier to understand each concept. Cons: Less context switching between tabs, may miss relationships.

C

Progressive Disclosure (Learn Then Explore)

Ch 2: Start with simple Derivative Visualizer, then unlock Backward Pass Debugger after interaction

Ch 3: Convergence Race first (simplest), then unlock Loss Landscape and Weight Updates as users adjust parameters

Pros: Guided learning, less cognitive load initially. Cons: Users may want advanced features immediately.
