## The problem: internal covariate shift

As training proceeds, the distribution of activations at each layer shifts because the weights of preceding layers change. The next layer must constantly re-adapt to this shifting distribution. This is **internal covariate shift** — it slows training and makes it sensitive to initialization.

## Batch Normalization

BatchNorm normalizes activations across the batch dimension at each layer, then applies learned scale ($\gamma$) and shift ($\beta$):

$$\mu = \frac{1}{N}\sum_{i=1}^N x_i, \quad \sigma^2 = \frac{1}{N}\sum_{i=1}^N (x_i - \mu)^2$$

$$\hat{x}_i = \frac{x_i - \mu}{\sqrt{\sigma^2 + \epsilon}}, \quad y_i = \gamma \hat{x}_i + \beta$$

During training, mean and variance are computed from the mini-batch. During evaluation, stored **running statistics** are used instead, giving deterministic output.

```cpp
// nn/BatchNorm.h — training mode
std::vector<float>& rm = running_mean;  // exponential moving average
std::vector<float>& rv = running_var;
for (int c = 0; c < C; ++c) {
    rm[c] = (1 - momentum) * rm[c] + momentum * batch_mean[c];
    rv[c] = (1 - momentum) * rv[c] + momentum * batch_var[c];
}
```

## BatchNorm backward

The backward pass through normalization is the subtlest chain rule in this framework. Three quantities depend on the batch mean and variance, so gradients flow through three paths back to the input.

Starting from $\frac{\partial L}{\partial \hat{x}_i}$ (gradient w.r.t. normalized input), the chain rule gives:

$$\frac{\partial L}{\partial \sigma^2} = \sum_i \frac{\partial L}{\partial \hat{x}_i} \cdot (x_i - \mu) \cdot \left(-\frac{1}{2}\right)(\sigma^2 + \epsilon)^{-3/2}$$

$$\frac{\partial L}{\partial \mu} = \sum_i \frac{\partial L}{\partial \hat{x}_i} \cdot (-\frac{1}{\sigma}) + \frac{\partial L}{\partial \sigma^2} \cdot \frac{-2}{N}\sum_i(x_i - \mu)$$

$$\frac{\partial L}{\partial x_i} = \frac{\partial L}{\partial \hat{x}_i} \cdot \frac{1}{\sigma} + \frac{\partial L}{\partial \sigma^2} \cdot \frac{2(x_i - \mu)}{N} + \frac{\partial L}{\partial \mu} \cdot \frac{1}{N}$$

The actual implementation precomputes `inv_std` and `inv_std_cubed` once per channel to avoid redundant sqrt operations, then computes dx_hat → dvar → dmean → dx in sequence:

```cpp
// From src/ops/batchnorm.cpp — backward pass
// Precompute per-channel inv_std and inv_std³
for (int c = 0; c < C; ++c) {
    float s = std::sqrt(batch_var[c] + eps);
    inv_std[c] = 1.0f / s;
    inv_std_cubed[c] = 1.0f / (s * s * s);
}

// dx_hat = grad * gamma;  also accumulate dgamma, dbeta
for (int n = 0; n < N; ++n)
    for (int c = 0; c < C; ++c) {
        dx_hat[n*C+c] = grad[n*C+c] * gamma[c];
        dgamma[c] += grad[n*C+c] * x_hat[n*C+c];
        dbeta[c]  += grad[n*C+c];
    }

// dvar: sum dx_hat * (x - mean) * (-0.5) * inv_std³
for (int n = 0; n < N; ++n)
    for (int c = 0; c < C; ++c)
        dvar[c] += dx_hat[n*C+c] * (x[n*C+c] - mean[c]) * (-0.5f) * inv_std_cubed[c];

// dmean: two terms (direct path + path through dvar)
for (int n = 0; n < N; ++n)
    for (int c = 0; c < C; ++c) {
        dmean[c] += dx_hat[n*C+c] * (-inv_std[c]);
        dmean[c] += dvar[c] * (-2.0f / N) * (x[n*C+c] - mean[c]);
    }

// dx: three terms
for (int n = 0; n < N; ++n)
    for (int c = 0; c < C; ++c)
        dx[n*C+c] = dx_hat[n*C+c] * inv_std[c]
                  + dvar[c] * 2.0f * (x[n*C+c] - mean[c]) / N
                  + dmean[c] / N;
```

If you implement BatchNorm and gradients go NaN, the backward pass is the first place to check — a wrong sign in the dvar term or missing the second dmean term will silently produce incorrect gradients that look plausible until training diverges.

## Train / eval mode

Modules with different training/evaluation behavior expose `train()` and `eval()` methods. `Sequential` propagates the mode recursively:

```cpp
// Always call model.eval() before inference
model.eval();
float acc = compute_accuracy(model, test_images, test_labels);
model.train();  // back to training mode
```

Forgetting `model.eval()` before inference means BatchNorm uses batch statistics from the inference batch (small, noisy) instead of the accumulated running statistics. Dropout would also randomly zero outputs during inference.

## Dropout

Dropout randomly zeroes each activation with probability $p$ during training, then scales surviving activations by $\frac{1}{1-p}$ to keep the expected value unchanged:

$$y_i = \begin{cases} \frac{x_i}{1-p} & \text{with probability } 1-p \\ 0 & \text{with probability } p \end{cases}$$

This is **inverted dropout** — the scaling happens during training, so at eval time the pass-through is an exact identity (no scaling needed).

```cpp
// nn/Dropout.h — training mode
float scale = 1.0f / (1.0f - p);
std::bernoulli_distribution dist(1.0f - p);
for (auto& m : mask)
    m = dist(gen) ? scale : 0.0f;
```

Why does this work? Each forward pass trains a different subset of the network. The network can't rely on any single neuron, forcing it to learn redundant representations — a form of regularization.
