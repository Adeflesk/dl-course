## Architecture

MiniGPT consists of:

1. **Token Embedding**: Convert character indices to dense vectors
2. **Positional Encoding**: Add position information (Chapter 10)
3. **Transformer Blocks** (2 layers): Attend to context and extract features
4. **Output Projection**: Map hidden state to vocab logits

Diagram:

```
Input tokens
     ↓
[Embedding] (token ID → embed_dim vector)
     ↓
[Add Positional Encoding]
     ↓
[TransformerBlock #1] (self-attention + FFN)
     ↓
[TransformerBlock #2] (self-attention + FFN)
     ↓
[LayerNorm]
     ↓
[Output Projection] (embed_dim → vocab_size logits)
     ↓
[Softmax] → next character probabilities
```

## Training: Teacher Forcing

During training, the model learns to predict the next character given the previous ones. We use **teacher forcing**:

Input: "The quick brown"
Target: " quick brown fox" (shift by 1)

At each position $t$, the model predicts token $t+1$ based on tokens $0:t$. The loss is the cross-entropy between predicted logits and ground truth:

$$\mathcal{L}_t = -\sum_v p_t(v) \log \hat{p}_t(v)$$

where $p_t$ is the true one-hot token and $\hat{p}_t$ is the predicted probability distribution.

Backprop through all transformer blocks updates all weights. The causal mask ensures each position only attends to earlier positions.

## Generation: Autoregressive Sampling

After training, the model generates text **autoregressively**:

1. Start with seed: "The "
2. Forward pass: embed "The ", compute logits for next token
3. Sample or take argmax to get next character "q"
4. Append "q", forward again
5. Repeat until max length

Two sampling strategies:

**Greedy (Argmax):**
```cpp
int next_token = argmax(logits);  // deterministic, often repetitive
```

**Temperature Sampling:**
```cpp
auto probs = softmax(logits / temperature);
int next_token = sample(probs);  // stochastic, varies with temperature
```

Lower temperature → distribution sharpens, output more deterministic.
Higher temperature → distribution flattens, output more random.

## C++ Implementation

The tensorflowccp framework provides everything needed:

```cpp
#include "nn/TransformerDecoder.h"

int main() {
    int vocab_size = 47;   // ~50 unique characters
    int embed_dim = 64;    // embedding dimension
    int num_heads = 4;     // 4 attention heads
    int num_layers = 2;    // 2 transformer blocks
    int max_seq_len = 256; // can handle up to 256 tokens

    // Build model
    nn::TransformerDecoder model(vocab_size, embed_dim, num_heads, num_layers, max_seq_len);
    nn::CrossEntropyLoss criterion;
    optim::Adam optimizer(model.parameters(), 0.001f);

    // Training loop
    for (int epoch = 0; epoch < num_epochs; ++epoch) {
        model.train();
        optimizer.zero_grad();

        // Forward pass
        Tensor logits = model.forward(input_tokens);  // [seq_len, vocab_size]
        Tensor loss = criterion.forward(logits, target_tokens);

        // Backward + optimize
        loss.backward(1.0f);
        optimizer.step();
    }

    // Generation
    model.eval();
    std::vector<int> seed_tokens = {char_to_idx['T'], char_to_idx['h'], char_to_idx['e']};
    std::vector<int> output = nn::generate_greedy(model, seed_tokens, 200);

    // Detokenize and print
    for (int idx : output) {
        std::cout << idx_to_char[idx];
    }
}
```

Key insight: **The entire pipeline is automatic differentiation**. You define forward(), call loss.backward(), and gradients flow through embedding → transformer blocks → output projection.

## What's Next? Real GPT-2

This capstone model is tiny (~100K parameters). Real language models are vastly larger:

- **GPT-2**: 1.5 billion parameters
- **GPT-3**: 175 billion parameters
- **LLaMA-70B**: 70 billion parameters

Scaling up involves:

1. **Larger corpus**: Wikipedia, Common Crawl (trillions of tokens)
2. **Larger model**: More layers, more heads, larger embeddings
3. **Efficient training**: Gradient checkpointing, mixed precision, distributed training
4. **Better tokenization**: Byte-pair encoding (BPE) reduces vocab size
5. **Post-training**: RLHF (reinforcement learning from human feedback) aligns the model with human preferences

Despite the scale difference, the fundamental architecture is identical to MiniGPT: embedding → transformer blocks → output projection.
