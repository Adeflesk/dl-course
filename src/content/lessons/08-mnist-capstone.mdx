MNIST is 60,000 handwritten digit images (28×28 pixels, 10 classes). It's the canonical benchmark for a working deep learning implementation. The goal here is 97%+ test accuracy using only the components built in this course.

## Architecture

```
[batch, 1, 28, 28]
  → Conv2d(1, 8, 5)     → [batch, 8, 24, 24]   # 8 kernels, 5×5
  → ReLU                → [batch, 8, 24, 24]
  → MaxPool2d(2)         → [batch, 8, 12, 12]   # halve spatial dims
  → Flatten              → [batch, 1152]
  → BatchNorm(1152)      → [batch, 1152]         # normalize, stabilize
  → Linear(1152, 64)    → [batch, 64]
  → ReLU                → [batch, 64]            # nonlinearity between linear layers
  → Dropout(0.3)        → [batch, 64]            # regularize
  → Linear(64, 10)      → [batch, 10]            # class logits
```

Total parameters: **76,954**.

**Note on the two linear layers:** A `ReLU` activation sits between `Linear(1152, 64)` and `Dropout`. Without it, the two linear layers would collapse into a single linear map during inference (since Dropout is identity at eval time), adding no nonlinear capacity. The `ReLU` ensures the 64-dimensional bottleneck is genuinely expressive.

```cpp
nn::Sequential model({
    std::make_shared<nn::Conv2d>(1, 8, 5),
    std::make_shared<nn::ReLU>(),
    std::make_shared<nn::MaxPool2d>(2),
    std::make_shared<nn::Flatten>(),
    std::make_shared<nn::BatchNorm>(1152),
    std::make_shared<nn::Linear>(1152, 64),
    std::make_shared<nn::ReLU>(),
    std::make_shared<nn::Dropout>(0.3f),
    std::make_shared<nn::Linear>(64, 10)
});
```

## Training setup

- **Loss:** CrossEntropyLoss (softmax + negative log likelihood, fused backward)
- **Optimizer:** Adam, lr=0.001
- **Batch size:** 100
- **Epochs:** 10
- **Data:** Full 60,000 training images, shuffled each epoch

```cpp
// From main.cpp — training loop
nn::CrossEntropyLoss criterion;
optim::Adam optimizer(model.parameters(), /*lr=*/0.001f);

for (int epoch = 0; epoch < 10; ++epoch) {
    model.train();
    shuffle(indices);  // random batch order each epoch

    for (int b = 0; b < num_batches; ++b) {
        Tensor images = get_batch(train_images, indices, b, batch_size);
        Tensor labels = get_batch(train_labels, indices, b, batch_size);

        // Forward
        Tensor logits = model.forward(images);
        Tensor loss = criterion.forward(logits, labels);

        // Backward
        model.zero_grad();
        loss.backward();

        // Update weights
        optimizer.step();
    }

    // Evaluate
    model.eval();
    float acc = compute_accuracy(model, test_images, test_labels);
    printf("Epoch %d  loss=%.4f  test_acc=%.1f%%\n", epoch, avg_loss, acc);
}

## What the network learned

The 8 kernels in Conv2d(1,8,5) learn to detect edges, corners, and curves. After pooling, the 1152-dimensional representation is a compressed spatial summary of those detections. The two linear layers map this summary to class probabilities.

BatchNorm after Flatten was critical — the 1152 features have very different scales coming out of the convolution. Without normalization, the linear layers struggle to find a consistent signal.

## CrossEntropy loss

For class $c$ and logits $z$:

$$L = -\log \frac{e^{z_c}}{\sum_j e^{z_j}} = -z_c + \log \left(\sum_j e^{z_j}\right)$$

The gradient with respect to logits $z_i$ is elegantly:

$$\frac{\partial L}{\partial z_i} = p_i - \mathbf{1}[i = c]$$

where $p_i = \text{softmax}(z)_i$. The loss pushes the correct class logit up and all others down, proportional to how much probability mass they incorrectly hold.

### Why fuse softmax and cross-entropy?

Computing softmax and then cross-entropy separately is numerically unstable — the log of a very small softmax probability underflows to $-\infty$. The framework fuses them: compute numerically stable log-softmax in one pass, store the softmax probabilities, produce a scalar loss — all in one operation:

```cpp
// From nn/CrossEntropyLoss.h — forward (per batch row)
float row_max = /* max over logits, for stability */;
float sum_exp = 0.0f;
for (int c = 0; c < num_classes; ++c) {
    softmax[r*C + c] = std::exp(logits[r*C + c] - row_max);
    sum_exp += softmax[r*C + c];
}
for (int c = 0; c < num_classes; ++c)
    softmax[r*C + c] /= sum_exp;              // normalize
total_loss -= std::log(softmax[r*C + target]); // NLL
```

The backward is elegantly simple — the softmax and log derivatives cancel, leaving:

```cpp
// From nn/CrossEntropyLoss.h — backward
// dL/dz_i = (softmax_i - 1{i == target}) / batch_size
for (int r = 0; r < batch_size; ++r)
    for (int c = 0; c < num_classes; ++c) {
        float indicator = (c == targets[r]) ? 1.0f : 0.0f;
        grad_logits[r*C + c] = (softmax[r*C + c] - indicator) * scale;
    }
```

The correct class logit gets gradient $(p_c - 1)/N$ — push it up. All other logits get $p_i/N$ — push them down, proportional to how much probability mass they incorrectly held.
